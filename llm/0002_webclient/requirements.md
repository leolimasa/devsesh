# Objective

To implement the web client for `devsesh`. See the README.md for the full project scope.

## Web Client

* Is written in React + Typescript.  [req.54k42f]
* Is embedded in the go binary using `embed.FS` and served by the server. [req.vdh4rk]
* Is responsive and can be accessed from both desktop and mobile devices. [req.9dmqy9]
* Uses shadcn/ui for UI components and overall design [req.damwg0]
* Dark mode by default [req.9k224a]

### Login

* User logs in by providing: [req.8udwqf]
  * an e-mail  [req.ybw3lj]
  * a passkey (FIDO2/WebAuthn) [req.5tsv4p]
* If there are no users, the user is prompted to create a new user [req.vgcps0]

### Create user

* User can create a new user by providing: [req.vqmg93]
  * an e-mail [req.xmzin4]
  * a passkey (FIDO2/WebAuthn) [req.zgkk62]

### Pairing

* User can pair their account by logging in and then providing a pairing code generated by the `devsesh login` command [req.29evhg]
* Once the pairing is complete, the command line client will be automatically granted access [req.xm73nc]

### Passkey management

* Adds a new webauthn/FIDO2 passkey to the current user's account for authentication [req.aqvd4y]
* Allows removing existing passkeys from the current user's account [req.67jse7]
* For logged in users only [req.jzlryq]

### Dashboard

* Display a list of active sessions for the current user with the following information: [req.f8zgi8]
  * Session ID [req.4eaaqa]
  * Start time [req.h2j29f]
  * Last ping time [req.mnocgd]
  * Status (active/inactive) [req.67f9op]
  	* Session is inactive when there is no ping for more than 5 minutes [req.kebcow]
  * Any additional metadata from the session json (e.g. project name, branch name, etc.) [req.85nmq5]
* Clicking the session will open a dedicated session details page [req.ncy560]
* The dashboard is updated in real-time as new sessions are started, pinged, ended, and updated [req.4lceak]
* Button to remove ALL staled sessions that have not been pinged for at least one hour [req.9s0lei]
* For logged in users only [req.xizyqy]

### Session details

* Displays all session metadata, updated in real time [req.gb3lku]
* Leave a placeholder which will display the session terminal in the future [req.h1hqvu]
